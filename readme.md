# Технічне завдання: "My Board Games Collection" (Повна версія)

## 1. Архітектура та Технології

- **Стек:** Vanilla HTML5, CSS3, JavaScript (ES6 Modules). Без фреймворків.
- **Структура коду (MVC):** Фрагментація на модулі для чистоти:
  - `database.js` (Дані)
  - `ui.js` (Локальна логіка відмальовування HTML)
  - `script.js` (Головний контролер, ініціалізація, події)
- **Формат:** Single Page Application (SPA). Динамічний рендеринг без перезавантаження.

## 2. База Даних (JSON структура)

- **Основні поля:** ID (транслітом для сортування), Назва, Статистика (гравці, час), Мова (англійською, напр. "Language Independent"). _Примітка: поле категорій прибрано за непотрібністю._
- **Контент:** Короткий опис для карток, масив абзаців для повного опису українською мовою.
- **Медіа:** Шляхи до обкладинки, PDF-правил та масив фотографій для візуалізації геймплею.
- **Доповнення:** Масив об'єктів з прапорцем `owned` (в наявності / у планах).

## 3. Інтерфейс та Дизайн (UI/UX)

- **Базова стилістика сайту:** Крафтово-картонна тематика, пастельно-бежева та коричнева палітра, безшовна текстура картону.
- **Індивідуальна темізація ігор (Кіллер-фіча):** Каркас (шрифти, розташування об'єктів) залишається незмінним для інтуїтивного UX. Але завдяки полю `themeClass`, кожна сторінка гри кардинально змінює візуал: текстуру фону, кольорову гаму, стиль кнопок під атмосферу конкретної гри.
- **Галерея:** Виведення кількох фотографій геймплею на сторінці гри.

## 4. Головна сторінка (Каталог)

- **Два формати відображення:**
  - "Магазин" (Плитка з обкладинками) — _Реалізовано_.
  - "Полиця" (Вид збоку, корінці коробок). Має враховувати пропорційно реальні розміри ігор (ширину/висоту).
- **Пошук та Фільтрація:** Живий пошук (Live Search) за оригінальною назвою та коротким описом.
- **Сортування:** Алфавітне. Завдяки використанню ID, ігри латиницею та кирилицею сортуються коректно з їх відповідниками (р=r).

## 5. Детальна сторінка гри

- **Макет:** Двоколонковий (зліва велика обкладинка, справа повноцінний опис і кнопка правил).
- **Блок доповнень:** Ігри в наявності відображаються повноцінно. Ігри з вішлиста (яких ще немає) — сірі, без додаткових фото.

## 6. Backlog (Ідеї для майбутніх версій)

- **3D Занурення (Кінематографічний перехід):** При кліку на гру, кришка коробки анімовано відкривається, і користувач "занурюється" всередину гри. _Складність:_ потребує параметризації (додавання в БД габаритів та типу коробки: звичайна кришка, магнітний клапан, висувна тощо).
- **Фоновий інтерактив:** `<canvas>` на задньому плані з фізикою (наприклад, кульки чи інші елементи).
- **3D Tilt-ефект:** Обкладинка гри на детальній сторінці реагує на рух курсору (нахиляється).
- **Вкладки (Tabs):** Опціональна можливість робити вкладки для ігор, де контенту занадто багато (на противагу Румікубу, де вони не потрібні).
